# 架构决策记录 (ADR) 索引

本目录包含 fafafa.ssl 项目的架构决策记录。

## 什么是 ADR？

架构决策记录 (Architecture Decision Record) 是一种轻量级的文档形式，用于记录重要的架构决策及其背景。

每个 ADR 包含：
- **状态**：提议、已接受、已废弃、已取代
- **上下文**：决策的背景和问题描述
- **决策**：做出的决定及其理由
- **后果**：决策带来的正面和负面影响

## 创建新 ADR

1. 复制模板 `ADR-template.md`
2. 按顺序编号（ADR-NNN）
3. 填写各节内容
4. 更新本索引

## ADR 模板

```markdown
# ADR-NNN: [标题]

## 状态
提议 | 已接受 | 已废弃 | 已取代 (日期)

## 上下文
[描述问题背景]

## 决策
[描述做出的决定]

## 后果
### 正面
[正面影响]

### 负面
[负面影响]

### 风险
[潜在风险]

## 替代方案
[考虑过的其他方案及否决理由]

## 参考
[相关文件、链接]
```
